## Phase 5: AI V2 Implementation (Apr 18 ~14:32)

The big one. Complete rewrite of enemy AI and squad management:

- **Encounter Director** — 4-phase combat flow: SETUP (5s deploy) -> ENGAGE (3 slots) -> PUSH (4 slots) -> FALLBACK (2 slots). Phases shift based on player HP and movement.
- **Engagement Slots** — Max 3 enemies shoot simultaneously. Others do meaningful waiting (reposition, reload, flank). 2s cooldown after releasing a slot.
- **Fireteam system** — Team 0 (frontal pressure, Heavy archetype) and Team 1 (mobile flanking, Rusher archetype). Teammates cluster near each other in cover.
- **Cover scoring V2** — Raycast concealment (+20), peek-shoot feasibility (+12), physical height, fireteam clustering (+8). Scores stored as node metadata for debug display.
- **Bark system** — Text bubbles: "COVERING!", "PUSHING!", "FLANKING!", "GRENADE!", "MAN DOWN!" to telegraph AI intent.
- **Gun jam mechanic removed** — Decided it wasn't fun. Weapons still have the data field but it's unused.
- **Fireteam debug visualization** — Lines between teammates (blue=FT0, orange=FT1), colored pathlines per AI state.
- **Free camera (F5)** — Fly around the arena while enemies keep fighting. Essential for tuning AI.

## Phase 6: Cover System Rebuild (Apr 18 ~15:00–15:21)

Moved from procedural cover to a designer-friendly prefab workflow:

- **8 cover prefabs** — Crate, CrateSmall, Barrier, WallSegment, WallSegmentLarge, FileCabinet, Desk, Pillar. Each is a `.tscn` with mesh + collision + cover points.
- **Fixed tactical arena layout** — Hand-placed zones: player spawn area, two mid-field clusters for fireteams, flanking corridors, rear sniper positions. 338 cover points, 811 NavMesh polygons. Same layout every run for consistent AI tuning.
- **CoverBuilder @tool script** — Editor tool that scans all StaticBody3D in the scene, filters by height/size, samples edges at 1.5m intervals, and auto-generates cover points. Decouples cover from props entirely — place props, click "Build All", done.
- **Removed runtime NavMesh baking** — NavMesh is now pre-baked in editor.

## Architecture Decisions

| Decision | Rationale |
|---|---|
| Engagement slots over free-fire | Prevents the "firing squad" feel; creates natural combat rhythm |
| Pre-baked NavMesh over runtime | More predictable AI paths; faster scene load |
| CoverBuilder as @tool | Level designers place props freely; cover points are derived, not manual |
| Removed gun jam | Felt punishing rather than tense; replaced tension source with AI pressure |
| Fixed arena layout | Needed deterministic setup for AI tuning before going procedural again |
